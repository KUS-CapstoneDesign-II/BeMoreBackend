## 🎯 표준 업데이트 프롬프트 (범용)

```

## 업데이트 정보
- **버전**: v[버전 번호] (예: v1.3.0)
- **배포 날짜**: [YYYY-MM-DD]
- **변경 유형**: [Major/Minor/Patch/문서개선]

## 주요 변경사항
1. [변경사항 1]
2. [변경사항 2]
3. [변경사항 3]

## 업데이트 체크리스트
- [ ] 버전 배지 업데이트 (상단)
- [ ] Version & Tech Stack 섹션 업데이트
- [ ] Changelog 최신 버전 추가
- [ ] API 문서 업데이트 (필요시)
- [ ] Quick Links 업데이트 (필요시)
- [ ] Footer 버전 정보 업데이트
- [ ] 날짜 오류 확인 (2025-01 → 2025-11 등)

## 추가 요구사항
[특별히 강조할 내용이나 추가 지침]
```

## 🎯 작성 규칙

### 버전 번호 규칙 (Semantic Versioning)
- **Major (X.0.0)**: Breaking Changes, 큰 아키텍처 변경
- **Minor (x.Y.0)**: 새로운 기능 추가 (하위 호환)
- **Patch (x.y.Z)**: 버그 수정

### Changelog 작성 규칙
```markdown
### v[X.Y.Z] (YYYY-MM-DD)

**[카테고리 이모지] [카테고리명]**
- [변경사항 1]
- [변경사항 2]

[추가 설명이 필요한 경우]

**하위 호환성**: [유지/일부 변경/Breaking Changes]
```

### 카테고리 이모지
- 🎯 새로운 기능
- 🔧 개선사항
- 🐛 버그 수정
- 📊 데이터베이스
- 🔐 보안
- ⚡ 성능
- 📝 문서
- 🚨 긴급 수정
- ⚠️ Breaking Changes

## 🔄 업데이트 워크플로우

### 1. 준비 단계
```bash
# 현재 버전 확인
grep "Version Badge" README.md

# 최근 Changelog 확인
grep -A 10 "## 📝 Changelog" README.md | head -20
```

### 2. 업데이트 실행
- 위 프롬프트 중 하나를 선택
- 필요한 정보 입력
- Claude에게 전달

### 3. 검증 단계
```bash
# 버전 번호 일관성 확인
grep -E "v[0-9]+\.[0-9]+\.[0-9]+" README.md

# 날짜 형식 확인
grep -E "[0-9]{4}-[0-9]{2}-[0-9]{2}" README.md

# 타임라인 순서 확인
grep "###" README.md | grep "v1\."
```

### 4. 커밋
```bash
git add README.md
git commit -m "docs: update README to v[X.Y.Z]"
git push origin [branch]
```

## 💡 팁 & 주의사항

### ✅ Do's
- 버전 번호는 일관되게 유지
- 최신 정보는 상단에 배치
- Breaking Changes는 명확히 표시
- 날짜 형식 통일 (YYYY-MM-DD)
- 예시 코드는 실제 작동하는 것으로

### ❌ Don'ts
- 이전 버전 정보 삭제 금지
- 날짜 순서 뒤바뀜 금지
- 불필요한 기술 용어 남발
- 구버전 문서 링크 제거 금지
- 임의로 섹션 순서 변경 금지
